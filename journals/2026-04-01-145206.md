## Discussions and decisions

* Shiny binding type documentation should remain conservative and
  contract-based: emit a type note only when the generator can derive it
  precisely from the current binding implementation or known structured policy.
* Component-specific binding semantics such as `wa-carousel`'s 0-based index
  should be expressed through narrow structured policy keys, not ad hoc
  handwritten generated prose.
* The boolean-default cleanup should stay narrow: add `Default: FALSE.` only
  for boolean HTML attributes when upstream metadata omits a default, rather
  than inventing broader fallback defaults for other metadata gaps.
* Repo-wide handwritten `{styler}` and `{lintr}` passes on `tools/` and tool
  tests are worth running periodically during the cleanup pass because the
  generator/tooling layer has accumulated enough changes that targeted
  lint-only cleanup is no longer sufficient.

## Pitfalls, reminders, and pointers

* Generated package files and site/reference outputs should continue to be
  treated as generated artifacts. Fix generator inputs or templates, then
  regenerate; do not patch the generated outputs directly.
* For local rebuilds, `devtools::document()`, `devtools::test()`, and
  `./tools/build_site.R --quiet` should be run sequentially rather than in
  parallel when validating rendered docs after generator changes, otherwise
  stale output can be mistaken for a generator bug.
* The cleanup pass is still in progress. Resume with the next cleanup set
