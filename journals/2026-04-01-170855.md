## Discussions and decisions

* `inst/CITATION` is appropriate package source when the project needs
  citation text that differs from the default formatting derived from
  `DESCRIPTION`; it is not merely a pkgdown hack.
* User-facing long-form docs should be shared vignette-first sources with the
  website treated as a second rendering target from the same material.
* The maintainer-facing `Build Tools` document should remain article-only
  rather than being forced into the installed vignette surface.
* The deferred `shinylive` app stub belongs with documentation-site support
  material under `vignettes/`, not under `tools/`.

## Pitfalls, reminders, and pointers

* Installed vignette discovery only sees top-level files under `vignettes/`.
  Keeping user-facing shared sources under `vignettes/articles/` would not
  have produced real installed package vignettes.
* `tools::buildVignettes(dir = '.')` can emit temporary rendered `.html` files
  back into `vignettes/` during local validation. Treat those as build
  artifacts and do not keep them as source.
