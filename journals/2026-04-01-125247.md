## Discussions and decisions

* The generated argument-doc cleanup should stay conservative and metadata-led:
  use machine-readable type/default/enum data when upstream provides it, but do
  not invent stronger claims when upstream only gives prose.
* `String` is the preferred generated label over `Character string` for the
  wrapper argument-doc type surface.
* Terse or imperfect upstream prose should generally be preserved unless the
  change is a clear correctness fix such as normalizing boolean literals to
  `TRUE` / `FALSE`.
* The new generated `Shiny Bindings` section is useful enough to keep even
  though roxygen/pkgdown render it after `Value` rather than between
  `Arguments` and `Value`.

## Pitfalls, reminders, and pointers

* Generated package files and site/reference outputs should continue to be
  treated as generated artifacts. Fix generator inputs or templates, then
  regenerate; do not patch the generated outputs directly.
* For local rebuilds, `devtools::document()` and `./tools/build_site.R --quiet`
  should be run sequentially rather than in parallel when validating rendered
  site pages, otherwise stale site output can be mistaken for a generator bug.
* The cleanup pass is still in progress. Resume with the next cleanup set
